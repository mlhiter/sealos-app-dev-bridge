Sealos App Dev Bridge

Six compact logo directions for a local Chrome MV3 bridge that maps localhost provider tabs to captured Sealos Desktop profiles. The recommended app icon keeps the mark visible at toolbar size while the variants stay single-color and editable.

Switchboard Bridge

Recommended

localhost tab + Desktop profile

Two compact surfaces are joined by a central switch. It reads as a browser-extension utility first, with enough mass to survive 16px toolbar rendering.

Session Loop

SDK request and response

A continuous loop describes the Desktop SDK round trip. The endpoints stay explicit so it does not feel like a generic infinity mark.

Tab Port

local origin routing

A lower local tab connects to an upper profile node. This direction emphasizes origin and port selection more than profile storage.

Profile Stack

multi-cluster profile set

Layered profile cards make multi-cluster support visible. The active row gets a socket-like endpoint for the selected current tab.

Cluster Socket

environment identity

A small cluster frame drops into a local socket. This is the most infrastructure-forward option, useful if the project wants a Kubernetes-adjacent feel.

Safe Origin

local-only boundary

The shielded outline turns the safety contract into the mark: local origins only, explicit profile reuse, no hidden auth bypass posture.
